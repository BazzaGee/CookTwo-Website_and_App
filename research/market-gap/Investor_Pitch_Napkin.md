# Cupla — Investor Overview

## The Shared Food System for Couples

---

## The Problem

Every couple deals with the same three things, every single day:

1. **"What's for dinner?"** — The nightly argument. Decision fatigue. Staring into the fridge.
2. **"Did you already buy milk?"** — Double-buying. Wasted food. Wasted money.
3. **"I need 400 calories, you need 700"** — One wants to lose weight, the other wants to gain muscle. They cook together but need different portions. No app solves this.

---

## The Product

A self-reinforcing loop that never stops working:

```
  WHAT YOU HAVE          WHAT TO COOK          WHAT TO BUY
 ┌──────────┐         ┌──────────┐         ┌──────────┐
 │          │         │          │         │          │
 │  PANTRY  │────────▶│ AI MEALS │────────▶│ SHOPPING │
 │          │         │          │         │          │
 └──────────┘         └──────────┘         └──────────┘
       ▲                                          │
       │            "Add Missing                  │
       │             to List"                     │
       │                                          │
       └──────── "Move to Pantry" ◄───────────────┘
```

**Step 1 — Pantry.** Type what's in your kitchen. "Chicken, rice, spinach, yogurt." Both partners see it. Real-time.

**Step 2 — AI Meal.** Press one button. AI looks at your pantry + both partners' diets and goals → generates a meal. Every ingredient marked green (you have it) or orange (you need it). One tap adds the orange ones to your shopping list.

**Step 3 — Shopping.** Shared grocery list. Both partners see changes instantly — under 100 milliseconds. Partner A checks off milk at the store, Partner B sees it disappear at home. When you check something off, tap "Move to Pantry" and the cycle starts again.

**It never stops.** The more you use it, the smarter it gets.

## The Moat — One Meal, Two Plates

This is the feature no competitor has.

Partner A wants to lose weight. Partner B wants to build muscle. They want to cook **one meal together**, not two separate meals.

The AI generates **one shared cooking workflow** with **two distinct plating instructions**:

```
        SAME CHICKEN STIR-FRY
       ┌─────────────────────┐
       │   Cook together.    │
       │   One pan.          │
       │   One recipe.       │
       └────────┬────────────┘
                │
        ┌───────┴───────┐
        ▼               ▼
  ┌───────────┐   ┌───────────┐
  │ PARTNER A │   │ PARTNER B │
  │           │   │           │
  │ 4oz chicken   │ 6oz chicken
  │ over spinach   │ over full
  │ ½ cup rice │   │ 1 cup rice│
  │           │   │           │
  │ ≈ 450 cal │   │ ≈ 700 cal │
  │ lose fat  │   │ build     │
  │           │   │ muscle    │
  └───────────┘   └───────────┘
```

**They cook together. They eat together. They hit different goals.**

This works because both partners enter their body stats (weight, height, age, activity level, goal). The app calculates each person's exact calorie target using the Mifflin-St Jeor equation. Then the AI generates a single recipe with two plating outputs — same ingredients, different portions, different macros.

## The Profiles — Who Are You Cooking For?

Both partners set up their own profile:

- **Diet:** Vegetarian, Vegan, Omnivore, Keto, Paleo, Gluten Free — or custom
- **Allergies:** Multi-select or type your own
- **Body stats:** Weight, height, age, gender, activity level
- **Goal:** Lose weight / Maintain / Gain muscle
- **Output:** Auto-calculated daily calorie target per person

Example:
- Alex — 28F, 68kg, moderate activity, lose weight → **1,700 cal/day**
- Jordan — 30M, 82kg, active, gain muscle → **3,200 cal/day**

The AI uses both profiles to generate meals that work for both people simultaneously.
